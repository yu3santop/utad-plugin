## Context

Sarah is creating her first project. This Test Instance verifies that the
experience meets non-functional quality standards — speed, cognitive load,
and trust signals — regardless of whether the functional outcome is correct.

## GIVEN

- The user has just confirmed their email and landed on the welcome screen
- The user is on a standard broadband connection
- The user has not read any documentation or tutorials

## WHEN

The user completes the first-project creation flow from start to finish,
without any assistance

## THEN

### Performance
- The user sees a response to their action within 500 milliseconds
- The project is fully created and navigable within 1.5 seconds
- At no point does the user see a blank screen for more than 300 milliseconds
- If creation takes longer than 1 second, a visible progress indicator appears

### Cognitive load
- The user completes the action in no more than 3 decisions
- Every label and placeholder is written in plain language
- No field requires knowledge that a brand-new user would not have
- The primary action button is the most visually prominent element
  on the screen at every step

### Trust signals
- The newly created project is immediately visible in any navigation
  the user has already seen (e.g. sidebar, home screen)
- If the user navigates away and returns, the project is still there
- No confirmation email or additional step is required to access the project

### Accessibility
- The entire flow is completable using keyboard navigation only
  (Tab, Enter, and arrow keys)
- All interactive elements have visible focus indicators

## Acceptance Criteria

- [ ] Action response time under 500ms
- [ ] Full completion time under 1.5 seconds on standard broadband
- [ ] Progress indicator shown if completion exceeds 1 second
- [ ] No more than 3 decisions required from start to finish
- [ ] Project visible in navigation immediately after creation
- [ ] Flow completable by keyboard only
