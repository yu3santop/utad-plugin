## Context

Sarah is a project manager at a small agency. She has just signed up after
a colleague recommended the product. She is motivated and ready to get started,
but she has never used the product before and wants her first experience to
feel simple and confidence-building.

## GIVEN

- The user has just created an account and confirmed their email
- The user has not yet created any projects
- The user is on the welcome screen for the first time

## WHEN

The user creates their first project by entering a name and clicking to confirm

## THEN

### Functional result
- A new project appears in the user's project list with the name they entered
- The user is taken inside the new project, ready to start adding work
- The project shows a helpful starting point (e.g. a prompt or first step)

### Experience quality
- The project is created and the user lands inside it within 1 second
- The user does not need to fill in more than 2 fields to complete this action
- The next obvious action (e.g. "add your first task") is visible without scrolling

### Emotional result
- The user feels accomplished because they completed a real action in under
  30 seconds, not just browsed a tutorial

## Acceptance Criteria

- [ ] Project creation completes in under 1 second
- [ ] User lands inside the new project immediately after creation
- [ ] No more than 2 required fields to create a project
- [ ] A first-action prompt is visible on the empty project screen
- [ ] The project name entered by the user appears exactly as typed
