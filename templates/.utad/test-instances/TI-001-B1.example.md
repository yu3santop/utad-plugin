## Context

Same scenario as TI-001-A: Sarah is creating her first project.
She types a project name but accidentally leaves the field blank
and clicks to confirm.

## GIVEN

- The user has just created an account and confirmed their email
- The user is on the welcome screen for the first time
- The user has left the project name field empty

## WHEN

The user attempts to create a project without entering a name

## THEN

### What the user sees
- The action does not complete
- The name field is clearly highlighted to show it needs attention
- A short, friendly message explains that a name is needed
  (not a generic error code or system message)

### Recovery path
- The user's cursor is placed directly in the name field, ready to type
- The user can fix the problem and complete the action without reloading
- No data the user may have entered in other fields is lost

### Emotional result
- The user feels mildly inconvenienced but not confused or alarmed
- The user's trust in the product is maintained because the guidance
  is clear and recovery is immediate

## Acceptance Criteria

- [ ] An empty name field prevents project creation
- [ ] The error message uses plain language (no error codes or technical terms)
- [ ] Focus returns to the name field automatically after the error
- [ ] Any other fields the user filled in are preserved after the error
- [ ] The user can complete the action immediately after fixing the name
